# 회원가입 플로우 (PROMPT 2 구현)

## 구현 완료된 기능

### 1. 수요기업 회원가입 (/signup/buyer)

**페이지 경로**: `/signup/buyer`

**입력 필드**:
- ✅ 회사명 (필수, 최대 255자)
- ✅ 사업자등록번호 (필수, XXX-XX-XXXXX 형식, 자동 하이픈)
- ✅ 지역 (필수, 17개 시/도 선택)
- ✅ 기업 규모 (필수, Q1-Q4)
- ✅ 담당자명 (필수, 최대 100자)
- ✅ 이메일 (필수, 유효성 검증)
- ✅ 연락처 (필수, 01X-XXXX-XXXX 형식, 자동 하이픈)

**주요 기능**:
- React Hook Form + Zod를 이용한 실시간 검증 (onChange 모드)
- 인라인 에러 메시지 (role="alert")
- 사업자등록번호/전화번호 자동 하이픈 추가
- 로딩 상태 및 중복 제출 방지
- 중복 사업자등록번호 체크 (예: "123-45-67890" 입력 시 409 에러 시뮬레이션)
- PII 마스킹 로깅 (전화번호, 이메일)
- 제출 성공 시 `/search`로 리다이렉트

### 2. SI 파트너 회원가입 (/signup/partner)

**페이지 경로**: `/signup/partner`

**섹션 구조**: Tabs로 분할 (기본 정보 / 시공 이력 / 역량)

#### Tab 1: 기본 정보
- 수요기업과 동일한 회사 정보 필드
- 담당자 정보 필드

#### Tab 2: 시공 이력
- ✅ 완료한 프로젝트 수 (필수, 0 이상 정수)
- ✅ 실패한 프로젝트 수 (필수, 0 이상 정수)
- ✅ **시공 성공률 자동 계산** 및 실시간 표시
  - 공식: `(완료 / (완료 + 실패)) × 100`
  - 소수점 1자리까지 표시
  - 총 프로젝트 0건일 때 → 0% 표시

#### Tab 3: 역량
- ✅ 기술 역량 태그 (필수, 최소 1개 ~ 최대 10개)
- ✅ 사전 정의된 태그 10개:
  - 용접, 조립, 도장, 검사, 팔레타이징
  - 픽앤플레이스, CNC 로딩, AGV, 협동로봇, 비전검사
- ✅ 사용자 정의 태그 직접 입력 가능
- ✅ Tag Input 컴포넌트:
  - Enter 키로 추가
  - X 버튼으로 제거
  - Backspace로 마지막 태그 삭제
  - 추천 태그 클릭으로 즉시 추가
  - 태그 개수 표시 (N/10)

**주요 기능**:
- 제출 성공 시 `/signup/partner/pending`으로 리다이렉트
- 모든 Zod 검증 규칙 적용
- 반응형 디자인 (모바일 단일 컬럼)

### 3. SI 파트너 심사 대기 페이지 (/signup/partner/pending)

**페이지 경로**: `/signup/partner/pending`

**표시 내용**:
- ✅ 가입 신청 접수 완료 메시지
- ✅ 예상 검토 기간: **2~3 영업일**
- ✅ 검토 프로세스 3단계 시각화:
  1. 서류 검토 (진행 중 표시)
  2. 이력 확인
  3. 승인 완료
- ✅ 고객 지원 연락처:
  - 이메일: partner@robotsi.com
  - 전화: 1588-0000
  - 운영 시간: 평일 09:00-18:00
- ✅ "홈으로" 및 "로그인" 버튼

## 기술 구현 세부사항

### Zod 스키마 (/lib/schemas/auth.ts)

재사용 가능한 스키마 구조:
```typescript
- companyBasicsSchema: 회사 기본 정보
- contactInfoSchema: 담당자 정보
- buyerSignupSchema: 수요기업 전체
- siPartnerSignupSchema: SI 파트너 전체
```

### 자동 포맷팅 함수 (/lib/formatters.ts)

```typescript
- formatBizRegistrationNo(): 사업자등록번호 하이픈 자동 추가
- formatPhoneNumber(): 전화번호 하이픈 자동 추가
- calculateSuccessRate(): 시공 성공률 계산
```

### Tag Input 컴포넌트 (/app/components/forms/TagInput.tsx)

**Features**:
- 키보드 네비게이션 (Enter, Backspace)
- 추천 태그 필터링 (입력값 기반)
- 중복 방지
- 최대 개수 제한
- 접근성 (aria-label)

## 접근성 (WCAG 2.1 AA)

✅ **구현된 접근성 기능**:
- 모든 입력 필드에 `<Label>` 연결
- 에러 메시지에 `role="alert"`
- 에러 발생 시 `aria-describedby` 및 `aria-invalid` 속성
- 키보드 네비게이션 지원
- 포커스 관리
- 의미론적 HTML

## 반응형 디자인

- **Mobile** (≤640px): 단일 컬럼, 전체 너비 입력 필드
- **Tablet** (641-1023px): 2컬럼 그리드 (지역/규모)
- **Desktop** (≥1024px): 최대 너비 640px, 중앙 정렬

## 폼 검증 타이밍

- **모드**: `onChange` (입력 중 실시간 검증)
- **에러 표시**: ≤200ms (인라인)
- **제출 버튼**: 검증 실패 시 disabled 상태

## 에러 핸들링

### 수요기업 회원가입
- **409 (중복)**: "이미 등록된 사업자등록번호입니다" (Toast)
- **400 (검증 실패)**: 필드별 인라인 에러 메시지
- **500 (서버 에러)**: "회원가입에 실패했습니다. 다시 시도해주세요" (Toast)

### SI 파트너 회원가입
- 동일한 에러 핸들링 패턴
- 성공 시: "회원가입 신청이 완료되었습니다!" (Toast)

## 테스트 시나리오

### 수요기업 회원가입 테스트

1. `/signup/buyer` 접속
2. 모든 필드를 유효한 값으로 입력
3. 사업자등록번호 "1234567890" 입력 → 자동으로 "123-45-67890" 변환 확인
4. 전화번호 "01012345678" 입력 → 자동으로 "010-1234-5678" 변환 확인
5. 제출 → `/search`로 리다이렉트 확인

**중복 테스트**:
- 사업자등록번호 "123-45-67890" 입력 → 중복 에러 토스트 확인

### SI 파트너 회원가입 테스트

1. `/signup/partner` 접속
2. **기본 정보** 탭에서 회사/담당자 정보 입력
3. **시공 이력** 탭으로 이동
   - 완료 프로젝트: 80
   - 실패 프로젝트: 20
   - 성공률 자동 계산: 80.0% 확인
4. **역량** 탭으로 이동
   - "용접" 추천 태그 클릭 → 추가 확인
   - 입력창에 "사출" 입력 후 Enter → 커스텀 태그 추가 확인
   - X 버튼으로 태그 제거 확인
   - 태그 개수 2/10 표시 확인
5. 제출 → `/signup/partner/pending`으로 리다이렉트 확인

### 검증 에러 테스트

1. 빈 폼 제출 시도 → 모든 필수 필드에 에러 메시지 표시 확인
2. 잘못된 이메일 형식 → "올바른 이메일 형식이 아닙니다" 확인
3. 사업자등록번호 10자리 미만 → "올바른 사업자등록번호 형식이 아닙니다" 확인
4. 역량 태그 0개 → "최소 1개의 역량 태그를 선택해주세요" 확인
